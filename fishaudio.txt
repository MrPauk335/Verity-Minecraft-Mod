2.  В API-запросе передать:

### 📝 Пошаговая инструкция

#### Шаг 1: Получите API-ключ
1.  Зарегистрируйтесь или войдите на [fish.audio](https://fish.audio).
2.  Перейдите в раздел **API Keys** (или Настройки → API).
3.  Создайте новый ключ и скопируйте его.

#### Шаг 2: Сформируйте API-запрос

Вот готовые примеры кода для генерации речи голосом Verity.

**Пример на Python (рекомендуется):**
```python
import requests

# Ваши данные
API_KEY = "ВАШ_API_КЛЮЧ"  # Замените на ваш ключ!
VOICE_ID = "b3c51bf6029f4201a342b40827250784"  # ID голоса Verity

url = "https://api.fish.audio/v1/tts"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json",
    "model": "s2.1-pro-free"  # Указываем бесплатную модель
}

data = {
    "text": "Привет! Я использую голос Verity через бесплатный API Fish Audio.",
    "reference_id": VOICE_ID,  # Указываем ID конкретного голоса
    "format": "mp3"  # Формат аудио (mp3, wav, opus, pcm)
}

response = requests.post(url, json=data, headers=headers)

if response.status_code == 200:
    with open("verity_speech.mp3", "wb") as f:
        f.write(response.content)
    print("✅ Аудио сохранено в verity_speech.mp3")
else:
    print(f"❌ Ошибка {response.status_code}: {response.text}")
```

**Пример через cURL (для теста в терминале):**
```bash
curl -X POST https://api.fish.audio/v1/tts \
  -H "Authorization: Bearer ВАШ_API_КЛЮЧ" \
  -H "Content-Type: application/json" \
  -H "model: s2.1-pro-free" \
  -d '{
    "text": "Привет! Я использую голос Verity через бесплатный API Fish Audio.",
    "reference_id": "b3c51bf6029f4201a342b40827250784",
    "format": "mp3"
  }' --output verity_speech.mp3
```

#### Шаг 3: Запустите код и получите аудио

### 🚨 Возможные проблемы и решения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| **401 Unauthorized** | Неверный или отсутствующий API-ключ | Проверьте, что в заголовке `Authorization` указан правильный ключ |
| **402 Payment Required** | Недостаточно кредитов (не для бесплатного тарифа) | Пополните баланс на Fish Audio |
| **400 Bad Request** | Неверный ID голоса или параметры | Проверьте, что `reference_id` указан правильно, а формат запроса корректен |
| **429 Too Many Requests** | Превышен лимит запросов | Снизьте частоту запросов или подождите |
| **500 Internal Server Error** | Проблема на стороне Fish Audio | Попробуйте повторить запрос через несколько минут |

### 💡 Рекомендации по использованию

1.  **Качество аудио**: Для максимального качества используйте `format: "wav"` или `format: "mp3"` с `mp3_bitrate: 192`.
2.  **Скорость**: Для диалоговых ассистентов используйте `speed: 1.1-1.2` (немного быстрее).
3.  **Потоковая передача**: Для реального времени используйте WebSocket вместо REST API.
4.  **Ограничения бесплатного тарифа**: 
    *   Нет жёсткого лимита на количество запросов 【turn0search17】
    *   Действует политика добросовестного использования 【turn0search17】
